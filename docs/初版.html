主从分层Agent自动化调度系统

无状态子执行单元 + 文件级全局持久化 + 独立外部监控告警 全闭环实现方案

方案核心定位

本方案针对传统Ralph Shell循环脚本的核心痛点，设计了一套「主Agent调度+无状态SubAgent执行+文件系统持久化+OpenClaw独立监控」的四层架构，实现了可追溯、抗崩溃、可远程值守、高扩展的自动化工作流引擎。

解决了传统循环脚本上下文耦合、故障卡死、状态不可视、无人值守不可控的核心问题，可直接落地于浏览器自动化、多账号运营、数据采集、批量任务处理等场景。

一、核心架构总览

核心设计理念

  • ● 调度与执行解耦：主Agent只负责调度与决策，SubAgent只负责原子任务执行，互不耦合
  • ● 无状态执行：SubAgent无内存上下文，执行即销毁，彻底解决上下文污染、崩溃无法重试问题
  • ● 文件级全局状态：以Markdown文件作为全局唯一真相源，全链路可追溯、可回滚、可人工干预
  • ● 独立监控兜底：OpenClaw监控与主进程完全解耦，实现真正的无人值守与远程状态感知

适用场景

  • ● 浏览器自动化与RPA流程（多账号运营、批量表单处理、数据采集）
  • ● 跨境电商平台自动化上架、订单处理、库存同步
  • ● 批量数据处理、文档转换、内容生成与分发
  • ● 需要7*24小时无人值守、可远程监控的自动化任务
  • ● 需要高容错、可重试、可追溯的生产级自动化流程

二、架构分层详细设计

1 主Agent调度层（核心大脑）

核心职责

整个系统的决策与调度核心，负责全局任务管理、子Agent生命周期管控、进度校验与规则执行，是唯一的任务入口与进度管理者。

  • 加载全局规则与持久化进度，初始化任务上下文
  • 将大任务拆解为原子化、可独立执行的子任务
  • 按调度规则分发子任务，启动对应SubAgent执行单元
  • 校验SubAgent执行结果，更新全局进度文件
  • 异常兜底处理：失败重试、超时终止、任务降级
  • 任务完成判定与全流程收尾工作
Harness规则设计（核心约束）

主Agent的行为完全由Harness规则文件约束，实现逻辑与配置解耦，无需修改代码即可调整调度策略。

核心配置项
  • 角色定义：主Agent的身份、核心目标、禁止行为
  • 工作流规则：任务拆解标准、执行顺序、优先级策略
  • 重试策略：单任务最大重试次数、重试间隔、失败降级规则
  • 超时规则：SubAgent执行超时时间、主循环心跳间隔
  • 校验规则：子任务完成的判定标准、结果校验逻辑
  • 异常处理规则：连续失败的处理逻辑、停滞告警阈值
设计核心亮点

主Agent不执行任何具体业务逻辑，只做调度与决策，彻底和执行层解耦；所有状态全部持久化到文件，即使崩溃重启，也可从断点继续执行，不会出现任务中断或重复执行。

2 SubAgent执行层（无状态工作单元）

核心设计原则

完全无状态、原子化执行、用完即毁。每个SubAgent只负责执行单一原子任务，启动时只读全局文件，执行完成后只写入结果日志，不保留任何内存上下文，执行完毕即销毁。

核心职责
  • 启动时读取全局memory.md、progress.md、sub-rules.md，获取执行所需的全部参数与规则
  • 严格按照规则执行单一原子任务（浏览器操作、接口请求、数据处理、文件操作等）
  • 输出标准化执行日志与结果，写入全局日志文件
  • 执行完成后，标记任务状态，触发主Agent进度更新
  • 执行异常时，输出标准化错误信息，不污染全局状态
设计核心优势
极致抗崩溃

单任务执行失败/崩溃，只会销毁当前SubAgent，不会影响主进程和其他任务，可直接重试

零上下文污染

无内存上下文，每个任务都是全新环境，彻底解决传统循环中上下文串扰、变量污染问题

无限可扩展

可并行启动多个SubAgent，支持分布式部署，新增任务只需新增对应SubAgent，无需修改主架构

关键约束

SubAgent禁止修改全局progress.md的核心进度，仅可写入执行日志与结果标记，进度更新权限完全收归主Agent，避免多进程并发写入冲突。

3 文件持久化层（全局唯一真相源）

本层是整个系统的状态中枢，所有Agent的行为、进度、规则全部以纯文本Markdown文件存储，不依赖任何数据库，轻量、可追溯、可人工编辑、全链路透明。

核心文件设计
progress.md 进度文件

全局任务进度的唯一凭证，采用YAML Frontmatter + Markdown正文结构，支持机器读取与人工编辑。

核心字段：当前阶段、已完成步骤、待执行步骤、失败任务记录、重试次数、最近执行时间、任务状态标记

memory.md 全局记忆文件

存储全局上下文、固定参数、环境变量、账号信息、全局约束规则，所有Agent共享只读。

核心内容：任务基础信息、环境配置、账号凭证、全局阈值、禁止行为清单

规则与日志文件
harness/ 规则目录

存储主Agent与SubAgent的所有行为规则，实现配置与代码完全解耦。

核心文件：main-rules.md（主Agent调度规则）、sub-rules.md（SubAgent执行规范）、validate-rules.md（结果校验规则）

logs/ 运行日志目录

动态更新的全链路运行日志，按日期/任务拆分，包含主Agent调度日志、SubAgent执行日志、错误日志。

核心作用：问题排查、执行追溯、监控数据来源、合规审计

文件读写规则（核心约束）
  • 唯一写入权限：progress.md的核心进度仅允许主Agent修改，SubAgent仅可追加日志与结果标记
  • 原子写入：所有文件修改采用「临时文件写入+替换原文件」的原子操作，避免文件损坏
  • 文件锁机制：主Agent写入时加排他锁，避免并发写入冲突
  • 版本备份：每次进度更新自动备份上一版本，支持一键回滚到任意历史节点

4 OpenClaw外部监控层（无人值守兜底）

本层是整套系统实现真正无人值守的核心，与主Agent进程完全解耦，独立运行，负责全链路状态监测、异常检测与手机端实时告警，解决了「人离开电脑，无法感知Agent是否正常运行」的核心痛点。

核心设计原则

独立进程、非侵入式、全链路监测、多渠道告警。不依赖主Agent的任何接口，仅通过文件系统与进程状态进行监测，即使主Agent完全崩溃，监控进程仍可正常运行并触发告警。

核心监测指标
进程存活监测
  • 主Agent进程是否正常运行
  • 进程CPU/内存占用是否异常
  • 是否出现进程僵死、崩溃退出
进度更新监测
  • progress.md是否在规定时间内有更新
  • 是否出现任务停滞、进度卡死
  • 任务完成度与预期是否匹配
异常日志监测
  • 错误日志关键词匹配（error、timeout、blocked）
  • SubAgent连续失败次数阈值告警
  • 规则违反行为监测
系统环境监测
  • 磁盘空间是否充足
  • 网络连通性是否正常
  • 浏览器/驱动环境是否正常
告警与联动能力
多渠道手机告警（核心能力）

支持国内主流推送渠道，实现手机端实时告警，人离开电脑也能随时掌握Agent运行状态。

支持渠道：微信推送（Server酱）、钉钉机器人、企业微信机器人、飞书机器人、短信告警、邮件告警

自动联动处置

除了告警，还可配置自动处置规则，实现故障自愈。

  • 主进程崩溃时，自动重启主Agent
  • 环境异常时，自动清理环境、重置浏览器
  • 连续失败时，自动回滚进度、暂停任务
  • 严重异常时，自动终止所有进程，避免风险扩大

三、完整工作流闭环

  1. 1
    系统初始化

    用户配置全局memory.md与harness规则文件，初始化progress.md空白进度文件，定义任务目标与执行规则。

  2. 2
    主Agent启动

    主Agent加载规则文件与进度文件，初始化任务上下文，校验环境与配置，进入主调度循环。

  3. 3
    任务拆解与分发

    主Agent根据当前进度，拆解出下一个原子子任务，生成执行指令与参数，启动对应的SubAgent执行单元。

  4. 4
    SubAgent无状态执行

    SubAgent启动后，读取全局文件获取执行参数与规则，执行原子任务，输出标准化执行日志与结果，执行完成后自动销毁。

  5. 5
    结果校验与进度更新

    主Agent等待SubAgent执行完成，根据校验规则验证执行结果，验证通过则更新progress.md进度，验证失败则按规则触发重试或降级。

  6. 6
    循环执行

    主Agent重复步骤3-5，直到所有任务完成，或触发终止规则。

  7. 7
    OpenClaw全流程独立监控

    从系统启动开始，OpenClaw独立进程持续监测进程状态、进度更新、异常日志，出现异常时立即推送告警到用户手机，或触发自动处置规则。

  8. 8
    任务收尾

    所有任务完成后，主Agent生成任务总结报告，备份全量日志与进度文件，推送任务完成通知到用户手机，正常退出。

四、与传统Ralph Shell循环方案对比

对比维度 传统Ralph Shell循环方案 本方案
架构设计 单层死循环，调度与执行耦合 主从分层架构，调度与执行完全解耦
状态管理 内存上下文，进程崩溃即丢失，易污染 文件级持久化，全局唯一真相源，断点续跑
抗崩溃能力 单步异常即卡死整个循环，无兜底 SubAgent异常不影响主进程，自动重试，故障自愈
可追溯性 仅基础日志，无结构化进度，无法回溯 全链路结构化进度+日志，每一步可追溯、可回滚
无人值守能力 无独立监控，卡死无法感知，无法远程值守 独立OpenClaw监控，手机实时告警，真正无人值守
可扩展性 新增任务需修改主循环，耦合度高，无法并行 新增任务只需新增SubAgent，支持并行与分布式扩展
可维护性 逻辑与配置耦合，修改规则需改代码 规则与代码完全解耦，配置化管理，无需改代码即可调整策略

2. 主Agent核心代码（main_agent.py）

import os
import time
import yaml
import logging
from datetime import datetime
from pathlib import Path
from sub_agents.base_subagent import SubAgentFactory

# 路径配置
BASE_DIR = Path(__file__).parent
DATA_DIR = BASE_DIR / "data"
HARNESS_DIR = BASE_DIR / "harness"
LOG_DIR = BASE_DIR / "logs"
BACKUP_DIR = DATA_DIR / "backup"

# 初始化日志
logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s",
    handlers=[
        logging.FileHandler(LOG_DIR / "main_agent.log", encoding="utf-8"),
        logging.StreamHandler()
    ]
)
logger = logging.getLogger("MainAgent")

class MainAgent:
    def __init__(self):
        self.task_name = ""
        self.task_config = {}
        self.progress = {}
        self.main_rules = {}
        self.current_step = 0
        self.is_running = False
        self._init_dirs()
        self._load_config()

    def _init_dirs(self):
        """初始化目录"""
        for dir_path in [DATA_DIR, HARNESS_DIR, LOG_DIR, BACKUP_DIR]:
            dir_path.mkdir(parents=True, exist_ok=True)

    def _load_config(self):
        """加载配置与进度文件"""
        # 加载进度文件
        progress_file = DATA_DIR / "progress.md"
        if not progress_file.exists():
            raise FileNotFoundError("进度文件 progress.md 不存在，请先初始化")
        
        # 解析YAML Frontmatter
        with open(progress_file, "r", encoding="utf-8") as f:
            content = f.read()
            if content.startswith("---"):
                _, frontmatter, body = content.split("---", 2)
                self.progress = yaml.safe_load(frontmatter)
                self.progress_body = body
            else:
                raise ValueError("进度文件格式错误，缺少YAML Frontmatter")
        
        # 加载主规则文件
        with open(HARNESS_DIR / "main-rules.md", "r", encoding="utf-8") as f:
            self.main_rules_content = f.read()
        
        # 加载全局记忆文件
        with open(DATA_DIR / "memory.md", "r", encoding="utf-8") as f:
            self.memory_content = f.read()
        
        # 初始化任务参数
        self.task_name = self.progress.get("task_name", "未知任务")
        self.current_step = self.progress.get("current_step", 0)
        self.total_steps = self.progress.get("total_steps", 0)
        logger.info(f"【主Agent】初始化完成，任务：{self.task_name}，总步骤：{self.total_steps}，当前步骤：{self.current_step}")

    def _backup_progress(self):
        """备份进度文件"""
        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
        backup_file = BACKUP_DIR / f"progress_backup_{timestamp}.md"
        with open(DATA_DIR / "progress.md", "r", encoding="utf-8") as src:
            with open(backup_file, "w", encoding="utf-8") as dst:
                dst.write(src.read())
        logger.info(f"【主Agent】进度文件已备份：{backup_file.name}")

    def _update_progress(self, step, status, result=None, failed=False):
        """更新进度文件"""
        self._backup_progress()
        
        # 更新进度数据
        self.progress["current_step"] = step
        self.progress["last_update_time"] = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        self.progress["last_exec_result"] = result
        self.progress["current_status"] = status
        
        if status == "completed" and not failed:
            self.progress["completed_steps"].append(step)
            self.progress["success_count"] = self.progress.get("success_count", 0) + 1
        elif failed:
            self.progress["failed_steps"].append({"step": step, "result": result})
            self.progress["failed_count"] = self.progress.get("failed_count", 0) + 1
        
        # 原子写入进度文件
        new_content = f"---\n{yaml.dump(self.progress, allow_unicode=True)}---\n{self.progress_body}"
        temp_file = DATA_DIR / "progress.tmp"
        with open(temp_file, "w", encoding="utf-8") as f:
            f.write(new_content)
        temp_file.replace(DATA_DIR / "progress.md")
        
        self.current_step = step
        logger.info(f"【主Agent】进度已更新，当前步骤：{step}，状态：{status}")

    def _validate_result(self, step, result):
        """校验执行结果"""
        # 加载校验规则
        with open(HARNESS_DIR / "validate-rules.md", "r", encoding="utf-8") as f:
            validate_rules = f.read()
        
        # 简单校验逻辑，可根据规则扩展
        if result.get("success", False):
            logger.info(f"【主Agent】步骤{step}执行结果校验通过")
            return True
        else:
            logger.error(f"【主Agent】步骤{step}执行结果校验失败：{result.get('error', '未知错误')}")
            return False

    def _execute_step(self, step):
        """执行单一步骤"""
        logger.info(f"【主Agent】开始执行步骤{step}")
        max_retry = self.progress.get("max_retry_times", 3)
        retry_interval = self.main_rules.get("retry_interval", 5)
        
        for retry in range(max_retry):
            try:
                # 获取SubAgent实例
                sub_agent = SubAgentFactory.get_agent(step, self.memory_content, self.main_rules_content)
                # 执行任务
                result = sub_agent.execute()
                # 校验结果
                if self._validate_result(step, result):
                    return result, False
                else:
                    logger.warning(f"【主Agent】步骤{step}执行失败，重试次数：{retry+1}/{max_retry}")
                    time.sleep(retry_interval)
            except Exception as e:
                logger.error(f"【主Agent】步骤{step}执行异常：{str(e)}，重试次数：{retry+1}/{max_retry}")
                time.sleep(retry_interval)
        
        # 重试全部失败
        return {"success": False, "error": f"重试{max_retry}次全部失败"}, True

    def run(self):
        """启动主Agent调度循环"""
        logger.info(f"【主Agent】启动任务调度循环，任务：{self.task_name}")
        self.is_running = True
        self._update_progress(self.current_step, "running")
        
        try:
            while self.is_running and self.current_step < self.total_steps:
                next_step = self.current_step + 1
                # 执行步骤
                result, failed = self._execute_step(next_step)
                # 更新进度
                if not failed:
                    self._update_progress(next_step, "running", result)
                else:
                    # 处理失败，根据规则决定是否继续
                    stop_on_failure = self.main_rules.get("stop_on_failure", True)
                    if stop_on_failure:
                        self._update_progress(next_step, "failed", result, failed=True)
                        logger.error(f"【主Agent】步骤{next_step}执行失败，任务终止")
                        break
                    else:
                        self._update_progress(next_step, "skipped", result, failed=True)
                        logger.warning(f"【主Agent】步骤{next_step}执行失败，跳过继续执行")
                
                # 执行间隔，避免过快
                time.sleep(self.main_rules.get("step_interval", 2))
            
            # 任务完成
            if self.current_step >= self.total_steps:
                self._update_progress(self.current_step, "completed")
                logger.info(f"【主Agent】任务全部执行完成！任务：{self.task_name}")
        
        except KeyboardInterrupt:
            logger.warning("【主Agent】收到中断信号，任务暂停")
            self._update_progress(self.current_step, "paused")
        except Exception as e:
            logger.error(f"【主Agent】调度循环异常：{str(e)}", exc_info=True)
            self._update_progress(self.current_step, "failed", {"error": str(e)})
        finally:
            self.is_running = False

class BaseSubAgent(ABC):
    """SubAgent抽象基类，所有执行单元必须继承此类"""
    def __init__(self, step, memory_content, rules_content):
        self.step = step
        self.memory_content = memory_content
        self.rules_content = rules_content
        self.logger = logging.getLogger(f"SubAgent-Step{step}")
        self.exec_result = {"success": False, "step": step, "data": None, "error": None}

    @abstractmethod
    def _exec_logic(self):
        """核心执行逻辑，子类必须实现此方法"""
        pass

    def execute(self):
        """执行入口，统一封装异常处理与日志"""
        self.logger.info(f"【SubAgent-{self.step}】开始执行任务")
        try:
            # 执行核心逻辑
            result = self._exec_logic()
            self.exec_result["success"] = True
            self.exec_result["data"] = result
            self.logger.info(f"【SubAgent-{self.step}】任务执行成功")
        except Exception as e:
            self.exec_result["error"] = str(e)
            self.logger.error(f"【SubAgent-{self.step}】任务执行异常：{str(e)}", exc_info=True)
        finally:
            # 执行完成，返回结果，实例即将销毁
            return self.exec_result

class OpenClawMonitor:
    def __init__(self):
        self._load_config()
        self.alert_sender = AlertSender(self.config.get("alert", {}))
        self.last_progress_mtime = 0
        self.last_alert_time = {}
        self.main_pid = None
        logger.info("【OpenClaw】监控系统初始化完成")

    def _load_config(self):
        """加载监控配置"""
        with open(CONFIG_PATH, "r", encoding="utf-8") as f:
            self.config = yaml.safe_load(f)
        
        self.check_interval = self.config.get("check_interval", 10)
        self.progress_timeout = self.config.get("progress_timeout", 300)
        self.alert_cooldown = self.config.get("alert_cooldown", 60)
        self.monitor_items = self.config.get("monitor_items", {})

    def _check_process_alive(self):
        """检查主Agent进程是否存活"""
        if not self.monitor_items.get("process_alive", True):
            return True, None
        
        # 查找主Agent进程
        main_pid = None
        for proc in psutil.process_iter(['pid', 'name', 'cmdline']):
            try:
                cmdline = proc.cmdline()
                if 'main_agent.py' in cmdline and 'python' in proc.name():
                    main_pid = proc.pid
                    break
            except (psutil.NoSuchProcess, psutil.AccessDenied, psutil.ZombieProcess):
                continue
        
        self.main_pid = main_pid
        if not main_pid:
            return False, "主Agent进程未运行，可能已崩溃退出"
        
        # 检查进程状态
        try:
            proc = psutil.Process(main_pid)
            if proc.status() in [psutil.STATUS_ZOMBIE, psutil.STATUS_DEAD]:
                return False, "主Agent进程已僵死/死亡"
            
            # 检查CPU/内存占用是否异常
            cpu_percent = proc.cpu_percent(interval=1)
            memory_percent = proc.memory_percent()
            if cpu_percent > self.config.get("cpu_threshold", 90):
                return False, f"主Agent进程CPU占用异常：{cpu_percent}%"
            if memory_percent > self.config.get("memory_threshold", 80):
                return False, f"主Agent进程内存占用异常：{memory_percent}%"
            
            return True, f"主Agent进程正常运行，PID：{main_pid}，CPU：{cpu_percent}%，内存：{memory_percent}%"
        
        except Exception as e:
            return False, f"检查主Agent进程异常：{str(e)}"

    def _check_progress_update(self):
        """检查进度是否正常更新"""
        if not self.monitor_items.get("progress_update", True):
            return True, None
        
        progress_file = DATA_DIR / "progress.md"
        if not progress_file.exists():
            return False, "进度文件不存在，任务可能未初始化"
        
        # 获取文件修改时间
        mtime = progress_file.stat().st_mtime
        now = time.time()
        
        # 首次运行记录时间
        if self.last_progress_mtime == 0:
            self.last_progress_mtime = mtime
            return True, "首次监控，记录进度文件初始修改时间"
        
        # 检查是否超时未更新
        if now - mtime > self.progress_timeout:
            # 解析进度文件，检查是否已完成
            with open(progress_file, "r", encoding="utf-8") as f:
                content = f.read()
                if content.startswith("---"):
                    _, frontmatter, _ = content.split("---", 2)
                    progress = yaml.safe_load(frontmatter)
                    if progress.get("current_status") in ["completed", "failed"]:
                        return True, f"任务已结束，状态：{progress.get('current_status')}"
            
            return False, f"进度文件已{int(now - mtime)}秒未更新，任务可能已卡死"
        
        # 检查进度是否正常推进
        if mtime > self.last_progress_mtime:
            self.last_progress_mtime = mtime
            return True, "进度文件正常更新，任务运行中"
        
        return True, "进度文件未更新，但未超过超时阈值"

    def _check_error_log(self):
        """检查错误日志"""
        if not self.monitor_items.get("error_log", True):
            return True, None
        
        error_log_file = LOG_DIR / "error.log"
        if not error_log_file.exists():
            return True, "错误日志文件不存在，暂无错误"
        
        # 检查最近的错误日志
        try:
            with open(error_log_file, "r", encoding="utf-8") as f:
                lines = f.readlines()
                if not lines:
                    return True, "错误日志为空，暂无错误"
                
                # 检查最近5分钟内的错误
                now = datetime.now()
                recent_errors = []
                for line in lines[-20:]:  # 检查最近20行
                    try:
                        log_time_str = line.split(" - ")[0]
                        log_time = datetime.strptime(log_time_str, "%Y-%m-%d %H:%M:%S,%f")
                        if now - log_time < timedelta(minutes=5):
                            recent_errors.append(line)
                    except:
                        continue
                
                if recent_errors:
                    return False, f"最近5分钟内检测到{len(recent_errors)}条错误日志：{recent_errors[-1][:200]}"
                
                return True, "错误日志无新增异常"
        
        except Exception as e:
            return False, f"检查错误日志异常：{str(e)}"

    def _send_alert(self, alert_type, message):
        """发送告警，带冷却时间"""
        now = time.time()
        last_time = self.last_alert_time.get(alert_type, 0)
        if now - last_time < self.alert_cooldown:
            logger.warning(f"【OpenClaw】告警冷却中，跳过本次告警：{alert_type}")
            return
        
        logger.error(f"【OpenClaw】发送{alert_type}告警：{message}")
        success = self.alert_sender.send_alert(f"【Agent自动化系统告警】{alert_type}", message)
        
        if success:
            self.last_alert_time[alert_type] = now

    def run_check(self):
        """执行一次全量检查"""
        logger.info("【OpenClaw】开始执行监控检查")
        
        # 1. 进程存活检查
        process_ok, process_msg = self._check_process_alive()
        if not process_ok:
            self._send_alert("进程异常", process_msg)
        else:
            logger.info(f"【OpenClaw】进程检查通过：{process_msg}")
        
        # 2. 进度更新检查
        progress_ok, progress_msg = self._check_progress_update()
        if not progress_ok:
            self._send_alert("进度停滞", progress_msg)
        else:
            logger.info(f"【OpenClaw】进度检查通过：{progress_msg}")
        
        # 3. 错误日志检查
        log_ok, log_msg = self._check_error_log()
        if not log_ok:
            self._send_alert("错误日志异常", log_msg)
        else:
            logger.info(f"【OpenClaw】日志检查通过：{log_msg}")
        
        logger.info("【OpenClaw】监控检查执行完成")

    def start(self):
        """启动监控循环"""
        logger.info("【OpenClaw】启动监控循环，检查间隔：{}秒".format(self.check_interval))
        try:
            while True:
                self.run_check()
                time.sleep(self.check_interval)
        except KeyboardInterrupt:
            logger.info("【OpenClaw】收到中断信号，监控系统退出")
        except Exception as e:
            logger.error(f"【OpenClaw】监控循环异常：{str(e)}", exc_info=True)

if __name__ == "__main__":
    monitor = OpenClawMonitor()
    monitor.start()

import requests

class AlertSender:
    def __init__(self, alert_config):

        # 钉钉机器人
        if self.config.get("dingtalk", {}).get("enable", False):
            self.dingtalk_webhook = self.config["dingtalk"].get("webhook")
            self.dingtalk_secret = self.config["dingtalk"].get("secret")
            self.channels.append("dingtalk")
            logger.info("告警渠道已启用：钉钉机器人")
        
        # 企业微信机器人
        if self.config.get("wecom", {}).get("enable", False):
            self.wecom_webhook = self.config["wecom"].get("webhook")
            self.channels.append("wecom")
            logger.info("告警渠道已启用：企业微信机器人")
        
        logger.info(f"告警渠道初始化完成，共启用{len(self.channels)}个渠道")

    def _send_serverchan(self, title, content):
        """发送Server酱微信推送"""
        if not self.serverchan_key:
            return False
        try:
            url = f"https://sctapi.ftqq.com/{self.serverchan_key}.send"
            data = {
                "title": title,
                "desp": f"{content}\n\n告警时间：{datetime.now().strftime('%Y-%m-%d %H:%M:%S')}"
            }
            resp = requests.post(url, data=data, timeout=10)
            if resp.status_code == 200 and resp.json().get("code") == 0:
                logger.info("Server酱微信推送发送成功")
                return True
            else:
                logger.error(f"Server酱推送失败：{resp.text}")
                return False
        except Exception as e:
            logger.error(f"Server酱推送异常：{str(e)}")
            return False

    def _send_dingtalk(self, title, content):
        """发送钉钉机器人消息"""
        if not self.dingtalk_webhook:
            return False
        try:
            # 签名处理（如果配置了secret）
            timestamp = str(int(datetime.now().timestamp() * 1000))
            sign = ""
            if self.dingtalk_secret:
                import hmac
                import hashlib
                import base64
                secret_enc = self.dingtalk_secret.encode('utf-8')
                string_to_sign = f"{timestamp}\n{self.dingtalk_secret}"
                string_to_sign_enc = string_to_sign.encode('utf-8')
                hmac_code = hmac.new(secret_enc, string_to_sign_enc, digestmod=hashlib.sha256).digest()
                sign = base64.b64encode(hmac_code).decode('utf-8')
            
            url = self.dingtalk_webhook
            if sign:
                url += f"×tamp={timestamp}&sign={sign}"
            
            data = {
                "msgtype": "markdown",
                "markdown": {
                    "title": title,
                    "text": f"### {title}\n\n{content}\n\n**告警时间**：{datetime.now().strftime('%Y-%m-%d %H:%M:%S')}"
                }
            }
            resp = requests.post(url, json=data, timeout=10)
            if resp.status_code == 200 and resp.json().get("errcode") == 0:
                logger.info("钉钉机器人消息发送成功")
                return True
            else:
                logger.error(f"钉钉推送失败：{resp.text}")
                return False
        except Exception as e:
            logger.error(f"钉钉推送异常：{str(e)}")
            return False

    def _send_wecom(self, title, content):
        """发送企业微信机器人消息"""
        if not self.wecom_webhook:
            return False
        try:
            data = {
                "msgtype": "markdown",
                "markdown": {
                    "content": f"### {title}\n\n{content}\n\n**告警时间**：{datetime.now().strftime('%Y-%m-%d %H:%M:%S')}"
                }
            }
            resp = requests.post(self.wecom_webhook, json=data, timeout=10)
            if resp.status_code == 200 and resp.json().get("errcode") == 0:
                logger.info("企业微信机器人消息发送成功")
                return True
            else:
                logger.error(f"企业微信推送失败：{resp.text}")
                return False
        except Exception as e:
            logger.error(f"企业微信推送异常：{str(e)}")
            return False

    def send_alert(self, title, content):
        """发送告警到所有已启用的渠道"""
        if not self.channels:
            logger.warning("未启用任何告警渠道，无法发送告警")
            return False
        
        logger.info(f"开始发送告警，标题：{title}")
        success_count = 0
        
        # 发送到各个渠道
        if "serverchan" in self.channels:
            if self._send_serverchan(title, content):
                success_count += 1
        
        if "dingtalk" in self.channels:
            if self._send_dingtalk(title, content):
                success_count += 1
        
        if "wecom" in self.channels:
            if self._send_wecom(title, content):
                success_count += 1
        
        logger.info(f"告警发送完成，成功渠道数：{success_count}/{len(self.channels)}")
        return success_count > 0

七、风险兜底与进阶扩展

风险兜底方案

  • ● 文件锁与并发冲突：采用文件级排他锁，主Agent写入时禁止其他写入操作，避免并发冲突；所有写入采用临时文件替换机制，避免文件损坏。
  • ● 主Agent崩溃兜底：OpenClaw监控检测到主进程崩溃时，可配置自动重启主Agent，自动从progress.md断点继续执行，无需人工干预。
  • ● 任务回滚机制：每次进度更新自动备份历史版本，出现异常时可一键回滚到任意历史进度，重新执行。
  • ● 风控熔断机制：配置连续失败阈值，达到阈值时自动暂停任务，推送告警，避免频繁操作触发平台风控。
  • ● 紧急终止机制：OpenClaw监控可配置紧急终止规则，出现严重异常时自动终止所有相关进程，避免风险扩大。

进阶扩展方向

  • ● 分布式部署：将SubAgent部署到多台服务器/容器，实现分布式并行执行，主Agent通过消息队列分发任务，支持大规模任务处理。
  • ● 可视化管理面板：开发Web管理面板，实时展示任务进度、运行日志、监控数据，支持远程启停任务、修改配置、手动干预。
  • ● LLM深度集成：主Agent接入大模型，实现任务自动拆解、异常自动排查、规则动态调整，实现真正的智能Agent调度。
  • ● 多租户支持：扩展为多租户系统，支持多个用户、多个任务同时运行，资源隔离，权限管控。
  • ● Docker容器化部署：打包为Docker镜像，一键部署运行，环境隔离，避免依赖冲突，支持K8s编排。
